# Glossary

This is the **single source of truth for terminology** used across the project. Every
other document links here instead of re-defining terms. If you introduce a new concept,
define it here once.

| Term | Definition |
| --- | --- |
| **defib** | The project and the name of the single CLI binary (`defib`). Short for "defibrillator" — it keeps agent tasks alive and revives them after they die. |
| **Client** | The short-lived `defib` process a user runs in a terminal. It parses commands and talks to the Daemon over IPC. It holds no long-lived state. |
| **Daemon** | The long-lived background process (`defib daemon run`). It owns all Tasks, runs the scheduler, spawns Provider processes, performs Detection, and persists state. Exactly one Daemon runs per user account. |
| **Supervisor** | The logic *inside* the Daemon that drives a single Task through its lifecycle state machine. One Supervisor instance per active Task. |
| **Provider** | An external AI coding-agent CLI that defib drives, e.g. Claude Code or GitHub Copilot CLI. |
| **Adapter** | The defib code that knows how to drive one specific Provider (build commands, parse sessions, classify output). Implements the Provider interface. |
| **Task** | A single unit of work the user asked defib to supervise: a prompt/instruction plus provider, working directory, session settings, and retry policy. A Task persists until it reaches a terminal state or is removed. |
| **Attempt** | One execution of the Provider process for a Task. A Task has one or more Attempts (attempt 1, 2, …). Each Attempt has its own captured logs and an Outcome. |
| **Outcome** | The classified result of an Attempt. One of the Outcome Categories (`SUCCESS`, `RATE_LIMIT`, `QUOTA_EXHAUSTED`, `SESSION_LIMIT`, `TRANSIENT_ERROR`, `FATAL_ERROR`, `UNKNOWN`). Full semantics live in [detection.md](detection.md). |
| **Detection** | The process of turning an Attempt's exit code and captured output into an Outcome (and, when present, a Reset Time). |
| **Reset Time** | An absolute time, extracted from Provider output, at which a rate/quota limit is expected to clear (e.g. from a `Retry-After` header or a "resets at 3pm" message). |
| **Session** | The Provider's own persisted conversation/context for a Task, identified by a **Session Ref** (usually a session id). Resuming reuses the Session so the agent keeps its context. |
| **Session Ref** | The identifier defib uses to resume a Provider Session. Either pre-generated by defib and passed to the Provider, or parsed from Provider output. |
| **Resume** | Continuing an existing Task by starting a new Attempt that reuses the Provider's Session (via the Provider's native resume mechanism). |
| **Headless mode** | Running the Provider non-interactively (e.g. `claude -p`) with machine-readable output. The default mode; easiest to capture and classify. |
| **Interactive mode** | Running the Provider attached to a pseudo-terminal (PTY) for flows that require interactive input. |
| **Scheduler** | The Daemon component that decides *when* each waiting Task's next Attempt should run, based on Reset Time, Backoff, and policy Caps. |
| **Backoff** | An increasing delay between Attempts when no Reset Time is known. See the algorithm in [architecture.md](architecture.md#scheduling). |
| **Availability check / Probe** | An optional user-configured command (or a lightweight retry Attempt) used to determine whether credits/quota have become available again. |
| **Caps** | Policy limits that stop a Task from retrying forever: `max_attempts`, `deadline`, and `max_total_wait`. Defined in [configuration.md](configuration.md). |
| **IPC** | Inter-process communication between Client and Daemon: newline-delimited JSON over a Unix domain socket. Protocol in [architecture.md](architecture.md#ipc-protocol). |
| **State dir / Config dir / Runtime dir** | The on-disk locations for persisted state, user configuration, and the IPC socket respectively. Exact paths in [architecture.md](architecture.md#on-disk-layout). |
| **Recovery** | Restoring supervision of in-flight Tasks after a Daemon restart, terminal crash, or machine reboot. |
| **Terminal state** | A Task state from which no further Attempts occur: `SUCCEEDED`, `FAILED`, or `STOPPED`. |
